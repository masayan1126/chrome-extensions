# Rich Markdown Preview Chrome拡張機能 - 要件定義書

## 1. プロジェクト概要

### 1.1 目的
ユーザーがローカルファイルシステムからMarkdownファイルを選択し、カスタマイズ可能なテーマでリアルタイムプレビューできるChrome拡張機能を開発する。

### 1.2 ターゲットユーザー
- 開発者・エンジニア
- テクニカルライター
- ドキュメント作成者
- Markdownを日常的に使用するユーザー

---

## 2. 競合分析サマリー

### 2.1 主要競合製品

| 拡張機能 | ユーザー数 | 評価 | 強み | 弱み |
|---------|----------|------|------|------|
| Markdown Viewer (simov) | 300,000+ | 4.3★ | 30+テーマ、オープンソース | セットアップが複雑 |
| Markdown Preview Plus | 90,000+ | 4.0★ | ライブリロード、KaTeX | レンダリング不完全 |
| Markdown Reader | 50,000+ | 4.9★ | 高速、絵文字/図表対応 | テーマ選択が少ない |
| Local Markdown Viewer | - | - | ゼロコンフィグ | 機能が限定的 |
| MarkView | - | - | モダンUI、フォルダブラウザ | 新しく実績が少ない |

### 2.2 競合の共通課題
1. **初期設定の複雑さ** - file://へのアクセス許可設定が必要
2. **不完全なMarkdownサポート** - 脚注、カラーテキスト等が未対応
3. **プラットフォーム固有のバグ** - macOS/Windows/Linuxで挙動が異なる
4. **カスタムテーマの制限** - ユーザー定義のカラー設定が困難
5. **ファイルブラウジング** - ディレクトリ単位での管理が不便

### 2.3 差別化ポイント（本プロジェクト）
- **ユーザー定義カラーテーマ** - 任意のカラーコードでカスタマイズ可能
- **人気テーマのプリセット** - ワンクリックで適用
- **ディレクトリベースのファイル管理** - フォルダを選択して中のファイルを一覧表示
- **即時プレビュー** - ファイル選択と同時に表示

## 5. 非機能要件

### 5.1 パフォーマンス
| 項目 | 要件 |
|------|------|
| 初回読み込み | 1秒以内 |
| ファイル切り替え | 200ms以内 |
| 大容量ファイル | 10MB以上でも3秒以内にレンダリング |
| メモリ使用量 | 100MB以下 |

### 5.2 セキュリティ・プライバシー
| 項目 | 要件 |
|------|------|
| オフライン動作 | 完全オフライン対応、外部API不使用 |
| データ送信 | ユーザーデータの外部送信なし |
| 権限 | 必要最小限の権限のみ要求 |
| ストレージ | ローカルストレージのみ使用 |

### 5.3 互換性
| 項目 | 要件 |
|------|------|
| Chrome | バージョン88以上 |
| Manifest | V3準拠 |
| OS | Windows/macOS/Linux |
| 解像度 | 1280x720以上 |

### 5.4 アクセシビリティ
| 項目 | 要件 |
|------|------|
| キーボード操作 | 全機能をキーボードで操作可能 |
| スクリーンリーダー | ARIA属性の適切な使用 |
| カラーコントラスト | WCAG 2.1 AA準拠 |

## 10. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| File System Access APIの制限 | 高 | フォールバックとしてinput[type=file]を用意 |
| Manifest V3の制限 | 中 | 最新のベストプラクティスに従う |
| 大容量ファイルのパフォーマンス | 中 | 仮想スクロール、遅延レンダリング |
| クロスブラウザ互換性 | 低 | Chrome専用として割り切る |
