# 機能改善仕様書: 同名ファイルの複数タブ表示対応

> Issue: #10 同名のファイルをプレビューで開けない  
> 作成日: 2026-04-04  
> ステータス: 確定

---

## 1. 概要

### 背景

異なるディレクトリに同名のMarkdownファイル（例: `docs/README.md` と `src/README.md`）が存在するプロジェクトでは、
現状の rich-markdown-preview では片方しかタブで開けないという制約がある。

### 目的

異なるディレクトリに配置された同名ファイルを、同時に複数タブでプレビューできるようにする。

### 対象ユーザー

ローカルのMarkdownファイルを複数ディレクトリにわたって管理・参照するユーザー（自分自身）。

### 改善後に実現できること

- 異なるディレクトリの同名ファイルを同時に開き、並べて参照・比較できる
- サイドバーからのクリック開封・ドラッグ&ドロップの両操作で同名ファイルを区別して開ける
- ブラウザ再起動後もタブが正しく復元され、同名ファイルが混同されない

### 本仕様書の目的

本仕様書は実装の判断基準として活用する。受け入れ条件（Section 4）を満たすことで改善完了とみなす。

---

## 2. 現状の課題

### 2-1. サイドバーからの開封時の不具合

`src/features/tabs/useTabOpen.ts` の `openTab` 関数では、`file.path` のみで既存タブの重複チェックを行っている。
File System Access API 経由で取得したファイルのパスが完全な絶対パスを保証していない場合、
異なるディレクトリの同名ファイルが同一と判定され、既存タブにフォーカスが移るだけで新タブが開かない。

```ts
// 問題のあるコード（useTabOpen.ts:17）
const existingTab = tabsRef.current.find((t) => t.file.path === file.path);
if (existingTab) {
  setActiveTabId(existingTab.id);
  return; // ← 新タブが開かれない
}
```

### 2-2. ドラッグ&ドロップ時の不具合

`openDroppedFile` 関数では `file.name` のみで重複チェックしており、パス（ディレクトリ）情報を考慮していない。
そのため、異なるディレクトリから同名ファイルをドロップしても、最初に開いたタブにフォーカスが移るだけになる。

```ts
// 問題のあるコード（useTabOpen.ts:48）
const existingTab = tabsRef.current.find((t) => t.file.name === file.name && !t.file.handle);
if (existingTab) {
  setActiveTabId(existingTab.id);
  return; // ← 新タブが開かれない
}
```

### 2-3. 重複チェックロジックの不統一

| 関数 | 重複チェックの基準 |
|---|---|
| `openTab`（サイドバー） | `file.path` のみ |
| `openDroppedFile`（D&D） | `file.name` のみ |

ロジックが統一されておらず、どちらの操作でも同名ファイルを正しく区別できない状態になっている。

### 2-4. 再現手順

1. 異なるディレクトリに同名のMarkdownファイルを用意する（例: `dir1/README.md`, `dir2/README.md`）
2. サイドバーまたはディレクトリツリーから `dir1/README.md` を開く
3. 同様に `dir2/README.md` を開こうとする
4. **期待**: 別タブとして `dir2/README.md` が開く
5. **実際**: `dir1/README.md` のタブにフォーカスが移るだけで、`dir2/README.md` は開かれない

ドラッグ&ドロップの場合も同様の手順で再現する。

### 2-5. 現状の期待しない動作

- 同名ファイルを開こうとすると、既存タブにフォーカスが移るだけ
- 新タブとして追加されない
- ユーザーには何もフィードバックがなく、なぜ開けないかわからない

## 3. 要件

### 機能要件

| # | 要件 | 優先度 |
|---|---|---|
| F-1 | 異なるディレクトリに存在する同名ファイルを、それぞれ別タブとして開けること | 必須 |
| F-2 | サイドバーのクリック操作とドラッグ&ドロップの両方で F-1 が動作すること | 必須 |
| F-3 | 同一ファイル（同パス・同ハンドル）を再度開こうとした場合は既存タブにフォーカスを移すこと | 必須 |
| F-4 | 同名ファイルのタブラベルにディレクトリ名を付記して区別できること（例: `dir1/README.md`） | 必須 |
| F-5 | タブ数の上限は設けず、同名ファイルを無制限に開けること | 必須 |
| F-6 | ブラウザ再起動後のタブ復元でも同名ファイルが正しく区別されること | 必須 |
| F-7 | 異なるファイル名（非同名）の既存動作に影響を与えないこと | 必須 |

### 非機能要件

- タブラベルは「親ディレクトリ名（1階層のみ）/ファイル名」の形式で表示する（**同名ファイルが複数開かれている場合のみ**付記）
- 重複チェックのロジックを `openTab` と `openDroppedFile` で統一する

### 設計上の制約・決定事項

| 項目 | 決定内容 |
|---|---|
| **同一ファイルの判定基準** | サイドバー開封: `file.handle` の同一性（`isSameEntry()` API）を優先。`handle` がない場合は `file.path` で判定 |
| **D&Dファイルのパス** | ブラウザのFile APIはセキュリティ上パスを公開しないため、D&Dで開いたファイルは `file.name` + `file.size` + `file.lastModified` の組み合わせで同一性を判定する |
| **タブラベルのディレクトリ名** | `file.path` から親ディレクトリ名（最後の `/` より前の部分）を抽出する。`FileInfo` に `dirName` フィールドを追加する |
| **同名判定の実装場所** | `useTabs.ts` で `tabs` 一覧を監視し、同名ファイルの有無を `Map<fileName, count>` で管理。`TabItem` にフラグを渡す |
| **IndexedDB識別子** | `filePath`（完全パス）を保存し、復元時に `file.path` で同名ファイルを区別する |

## 4. 受け入れ条件（AC）

以下の条件をすべて満たした時点で本改善を完了とみなす。

| # | 条件 | 操作 | 期待結果 |
|---|---|---|---|
| AC-1 | サイドバーから同名ファイルを開く | `dir1/README.md` → `dir2/README.md` の順にサイドバーからクリック | 2つの別タブが作成される |
| AC-2 | D&Dで同名ファイルを開く | `dir1/README.md` → `dir2/README.md` の順にD&D | 2つの別タブが作成される |
| AC-3 | タブラベルの表示形式 | AC-1 または AC-2 実施後 | タブラベルが `dir1/README.md` / `dir2/README.md` の形式で表示される |
| AC-4 | 同一ファイルの重複防止 | すでに開いている `dir1/README.md` をもう一度開こうとする | 新タブは作成されず、既存タブにフォーカスが移る |
| AC-5 | タブ復元 | 同名ファイルを複数タブで開いた状態でブラウザを再起動 | 再起動後に同名タブが正しく復元され、内容が混同されない |
| AC-6 | 既存動作への影響なし | `a.md` と `b.md` など異なる名前のファイルを開く | 従来通りに別タブとして開ける |
| AC-7 | テスト | `npm run test` を実行 | 既存テストがすべてパスする |

## 5. 影響範囲

### 修正対象ファイル

| ファイル | 変更内容 |
|---|---|
| `src/features/tabs/useTabOpen.ts` | `openTab` / `openDroppedFile` の重複チェックロジックを統一・修正 |
| `src/features/tabs/tabSave.ts` | 同名ファイルを区別できる識別子で保存するよう修正 |
| `src/features/tabs/tabRestore.ts` | 復元時に同名ファイルを正しく区別できるよう修正 |
| `src/features/tabs/tabDb.ts` | IndexedDB の保存スキーマを確認し、識別子変更が必要な場合は修正 |
| `src/features/tabs/TabItem.tsx` | 同名ファイルが存在する場合のみ親ディレクトリ名をタブラベルに付記 |
| `src/features/tabs/useTabs.ts` | タブ一覧から同名ファイルの有無を判定するロジックを追加 |
| `src/shared/types/index.ts` | `FileInfo` / `OpenTab` 型の変更要否を検討（必要に応じて修正） |

### 修正方針

- **重複判定の基準（サイドバー）**: `file.handle.isSameEntry()` で同一性を判定。handle がない場合は `file.path` にフォールバック
- **重複判定の基準（D&D）**: `file.name` + `file.size` + `file.lastModified` の組み合わせで同一性を判定（パス取得不可のため）
- **ロジックの統一**: 上記判定を行う共通関数 `isSameFile()` を作成し、`openTab` / `openDroppedFile` の両方で使用する
- **`FileInfo` 型の拡張**: `dirName?: string` フィールドを追加し、`file.path` から親ディレクトリ名を格納する
- **タブラベルの動的変更**: `useTabs` でタブ一覧を監視し、同名ファイルが複数開かれている場合のみ `dirName` を表示する

### 非影響範囲

- `src/features/preview/` — プレビューレンダリングへの影響なし
- `src/features/sidebar/` — サイドバー側の変更は不要（ファイル選択の渡し方は変わらない）
- `src/features/review/` — レビューコメント機能への影響なし
