# Release Notes v1.2.2

## GitHub Releases 向け（技術者向け）

## v1.2.2

### 🐛 バグ修正

**同名ファイルを別タブで開けない問題を修正** (#10)

異なるディレクトリに存在する同名ファイル（例: `dir1/README.md` と `dir2/README.md`）を
同時に複数タブでプレビューできるようになりました。

#### 変更内容

- `FileSystemFileHandle.isSameEntry()` API を使った正確なファイル同一性判定に変更
- `directoryReader` でフルパスを正しく構築するよう修正（再帰呼び出し時の親パス引き継ぎ）
- ドラッグ&ドロップ時の重複チェックを `name + size + lastModified` 複合キーに変更
- 同名タブが複数開かれている場合のみ `parentDir/filename` 形式でラベルを表示
- タブ復元時も同名ファイルが正しく区別されるよう修正
- `useTabRestore` にクリーンアップ処理を追加し競合状態を防止

#### 影響範囲

| ファイル | 変更内容 |
|---|---|
| `src/features/tabs/useTabOpen.ts` | 重複チェックを `isSameEntry()` ベースに変更 |
| `src/features/tabs/TabBar.tsx` / `TabItem.tsx` | 同名タブのラベル表示改善 |
| `src/shared/utils/directoryReader.ts` | フルパス生成の根本修正 |
| `src/shared/utils/fileSystem.ts` | `getDirName()` ユーティリティ追加 |
| `src/features/tabs/useTabRestore.ts` | 競合状態対策 |

#### 既知の制限

- ドラッグ&ドロップで開いたファイルはブラウザの API 制約上ディレクトリ情報を取得できないため、
  同名 D&D ファイルが複数ある場合はラベルにディレクトリ名が付記されません。

## Chrome拡張ストア向け（一般ユーザー向け）

【バージョン 1.2.2】

■ バグ修正

・同名のファイルを別タブで開けない問題を修正しました

  これまで、異なるフォルダに同じ名前のファイル（例：「メモ.md」）が
  複数ある場合、2つ目以降のファイルをタブで開けないことがありました。

  今回のアップデートで、フォルダが違えば同じ名前のファイルでも
  それぞれ別タブとして開けるようになりました。

  また、同じ名前のファイルが複数開かれている場合は、
  タブのラベルに「フォルダ名/ファイル名」の形式で表示され、
  どちらのファイルか一目で区別できるようになっています。
