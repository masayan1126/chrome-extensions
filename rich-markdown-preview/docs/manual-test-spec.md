# Rich Markdown Preview 手動テスト仕様書

> 対象バージョン: 1.0.9
> 最終更新: 2026-03-10
> 対象ブラウザ: Google Chrome（最新安定版）

本仕様書には、Chrome 拡張機能の実環境でのみ確認可能なテスト項目のみを記載しています。
自動テスト可能な項目は `docs/browser-tests/` を参照してください。

---

## テスト準備

1. Chrome に Rich Markdown Preview 拡張機能をインストール済みであること
2. テスト用フォルダに複数の `.md` ファイルとサブフォルダ（5階層）を含むディレクトリを用意すること
3. 隠しファイル（`.hidden-note.md` など）をテスト用フォルダ内に配置すること
4. Mermaid記法を含むMarkdownファイルを用意すること

---

## 1. フォルダ管理（7件）

File System Access API のネイティブOSダイアログに依存するため自動化不可。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 1-1 | フォルダを開く | 「フォルダを開く」→ テスト用フォルダを選択 | .md / .markdown / .mdx / .mkd がツリー形式で表示される |
| 1-2 | ディレクトリツリーの展開 | サブフォルダのフォルダ名をクリック | 展開/折りたたみが切り替わる |
| 1-3 | 5階層までの表示 | 5階層の深いフォルダを開く | 最大5階層まで正しく表示される |
| 1-4 | 隠しファイルの非表示 | 隠しファイルがあるフォルダを開く | .で始まるファイルが非表示になっている |
| 1-5 | 隠しファイルの表示切替 | 隠しファイル表示トグルをクリック | 隠しファイルの表示/非表示が切り替わる |
| 1-6 | フォルダのリフレッシュ | リフレッシュボタンをクリック | 外部で追加・削除したファイルが反映される |
| 1-7 | ファイルの選択とプレビュー | ファイルツリーからMDファイルをクリック | タブで開きMarkdownがレンダリングされる |

## 2. Mermaidダイアグラム（2件）

dev server 環境ではMermaid SVGレンダリングが正常に動作しない可能性があるため手動確認が必要。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 2-1 | Mermaidダイアグラム | mermaid記法（```mermaid）を含むMDを開く | フローチャート・シーケンス図等がSVGでレンダリングされる |
| 2-2 | テーマに応じたMermaid | ダーク/ライトテーマで同じMermaid図を確認 | テーマの明暗に応じてMermaid図のテーマも切り替わる |

## 3. タブのドラッグ&ドロップ並べ替え（1件）

ネイティブD&Dイベントの正確な座標制御が自動化ツールでは困難。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 3-1 | タブの並べ替え | タブをドラッグして別の位置にドロップ | タブの順序が変更される |

## 4. ファイル同期 / エディタ連動（3件）

FileSystemFileHandle による変更監視は拡張機能コンテキスト + フォルダ選択が前提。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 4-1 | 外部エディタでの変更検知 | プレビュー中のファイルをVS Code等で編集・保存 | 1秒以内にプレビューが自動更新される |
| 4-2 | 連続的な変更 | 外部エディタで複数回保存 | 各保存後にプレビューが自動更新される |
| 4-3 | 別ファイルの変更 | 非アクティブタブのファイルを外部エディタで編集 | タブ切替時に最新の内容が表示される |

---

## 5. セッション復元（4件）

chrome.storage.local + IndexedDB + FileSystemHandle の組み合わせが必要。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 5-1 | フォルダの復元 | フォルダを開いた状態で拡張機能を閉じ、再度開く | 前回のフォルダが復元される（復元ボタン表示の場合あり） |
| 5-2 | タブの復元 | 複数タブを開いた状態で閉じ、再度開く | 前回のタブが復元される |
| 5-3 | 設定の復元 | テーマ・フォント等を変更後、閉じ再度開く | 全設定が復元される |
| 5-4 | 復元ボタンの表示 | セッション情報がある状態で開く | 復元ボタンが表示され、クリックでフォルダが復元される |

---

## 6. ドラッグ&ドロップ / OS連携（3件）

OSレベルのドラッグ操作は自動化ツールから発火できない。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 6-1 | ファイルのドロップ | OS上のMDファイルを拡張機能にD&D | 新しいタブで開かれプレビューされる |
| 6-2 | ドロップゾーンの表示 | ファイルをメインエリアにドラッグしてホバー | 青いドロップゾーン（「Markdownファイルをドロップ」）が表示される |
| 6-3 | 非MDファイルの処理 | .txt, .jpg 等をドロップ | エラーにならず適切に処理される |

---

## 7. コンテキストメニュー（5件）

サイドバーにファイルツリーが表示されている必要があり、フォルダ選択が前提。

| # | テスト内容 | 操作手順 | 期待結果 |
|---|-----------|---------|---------|
| 7-1 | ファイルの右クリックメニュー | サイドバーのファイルを右クリック | 「ファイルを開く」「パスをコピー」「ファイル名をコピー」が表示される |
| 7-2 | パスのコピー | 「パスをコピー」を選択 | 完全パスがクリップボードにコピーされる |
| 7-3 | ファイル名のコピー | 「ファイル名をコピー」を選択 | ファイル名がクリップボードにコピーされる |
| 7-4 | フォルダの右クリックメニュー | サイドバーのフォルダを右クリック | 「フォルダ名をコピー」「フォルダを開く/閉じる」が表示される |
| 7-5 | メニュー外クリックで閉じる | メニュー表示中にメニュー外をクリック | コンテキストメニューが閉じる |
